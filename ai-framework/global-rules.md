# Global Rules

- Keep the system simple.
- Avoid duplicated prompts, logic, workflows, and artifacts.
- Do not build future features until needed.
- Apply DRY: avoid duplicating logic, components, copy structures, validation rules, or workflow contracts unless duplication makes the result clearer and safer.
- Apply KISS: prefer the simplest implementation that satisfies the approved plan and quality bar.
- Apply YAGNI: do not add abstractions, dependencies, features, integrations, or future-proofing that the approved plan does not require.
- Separate planning from execution.
- Every skill must produce an explicit output used by the next skill.
- Keep AI context inside the smart zone.
- Prevent unnecessary changes.
- Prevent hallucinations and unsupported claims.
- Prevent overconfidence.
- Prevent dead code.
- Prevent fixing one issue while introducing regressions.
- Prevent vague outputs, wrong formats, and excessive explanation.
- Favor repeatability, observability, and controlled execution.
- Prefer existing project patterns over new abstractions.
- Do not freely modify unrelated files.
