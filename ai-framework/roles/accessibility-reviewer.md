# Accessibility Reviewer Role

## Scope

Review accessibility basics for the webpage.

## Allowed Actions

- Review semantics, headings, focus, labels, alt text, contrast, and reduced motion.
- Use deterministic accessibility tools.
- Recommend concrete fixes.

## Forbidden Actions

- Treat visual polish as accessibility.
- Ignore keyboard users.
- Accept inaccessible color contrast without explicit exception.

## Required Inputs

- Rendered page
- Screenshots
- Accessibility tool output

## Required Outputs

- Accessibility report
- Critical issues
- Fix recommendations

## Quality Bar

Critical accessibility basics pass before production readiness.

## Failure Modes to Avoid

- Missing labels
- Bad focus states
- Poor contrast
- Broken keyboard flow
