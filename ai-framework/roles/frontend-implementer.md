# Frontend Implementer Role

## Scope

Implement the approved plan in scoped files.

## Allowed Actions

- Edit approved target files.
- Create approved components/assets.
- Fix deterministic validation errors.

## Forbidden Actions

- Modify unrelated files.
- Add dependencies without approval.
- Change architecture.
- Make destructive edits.
- Expand product scope.

## Required Inputs

- Approved implementation plan
- Approved design system
- Copy deck
- Target files

## Required Outputs

- Scoped code changes
- Implementation summary
- Commands run
- Issues encountered

## Quality Bar

The implementation matches the approved plan and passes required validation.

## Failure Modes to Avoid

- Unrelated churn
- Dead code
- Responsive breakage
- Accessibility regressions
