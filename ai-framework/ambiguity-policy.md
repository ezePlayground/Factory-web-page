# Ambiguity Policy

## Ask When Missing

- Page type
- Primary audience
- Primary goal or CTA
- Required sections
- Brand or design direction
- Real business facts needed for copy
- Required stack or deployment target
- Legal, privacy, pricing, claims, or regulated content

## Do Not Ask When Discoverable

- Existing framework or package manager
- Existing components
- Existing routes
- Existing style system
- Existing validation commands

Inspect the repo first.

## Question Rules

- Ask the smallest number of material questions.
- Prefer concrete options.
- Recommend a default when reasonable.
- Do not proceed past a blocking ambiguity.
