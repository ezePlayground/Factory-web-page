# Deterministic Tools

Use deterministic tools before relying on AI judgment.

| Category | Examples | Output |
|---|---|---|
| Formatting | Prettier, Biome | Formatted files or report |
| Linting | ESLint, Stylelint | Lint report |
| Type checking | TypeScript | Type errors or pass |
| Tests | Unit/e2e framework | Test report |
| Build | Framework production build | Build report |
| Accessibility | axe, Lighthouse | A11y findings |
| Responsive | Browser screenshots | Viewport evidence |
| Links | Link checker | Broken link report |
| HTML/CSS | Validators | Markup/style report |
| SEO | Metadata/headings checks | SEO report |
| Performance | Lighthouse, bundle analyzer | Performance report |
| Images | Image optimizer | Asset size report |
| Security | Dependency audit, secret scan | Security report |
| Diff | Git status/diff | Scope report |

Tool output must be copied or summarized into the relevant QA artifact and `HANDOFF.md`.
