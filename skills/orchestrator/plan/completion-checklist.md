# Completion Checklist: Plan

Before using `summary.response.md`, confirm:

- [ ] Approved discovery brief is present.
- [ ] Page objective is clear.
- [ ] Success criteria are testable.
- [ ] Required questionnaire blockers are answered or explicitly marked unknown.
- [ ] Human-provided facts are separated from proposed decisions.
- [ ] Proposed IA is complete enough for implementation approval.
- [ ] Section-by-section content is defined or missing content flags are listed.
- [ ] `frontend-design` helper output is included when visual direction is needed.
- [ ] Design options, wireframes, or static mockup requirement are included when direction is not already visually approved.
- [ ] Static mockup approval status is stated when visual direction needs approval.
- [ ] Final or recommended design direction is stated and awaits human approval.
- [ ] Brand direction is captured.
- [ ] Component inventory is complete enough for implementation.
- [ ] Architecture definition is specific.
- [ ] Required functionality is listed.
- [ ] Technical constraints are captured.
- [ ] Accessibility, SEO, responsive, and performance requirements are included.
- [ ] Validation plan is defined.
- [ ] Assumptions requiring approval are listed.
- [ ] Approved execution scope is ready for human approval.
- [ ] Summary asks the human to approve, modify, or reject.
