# Completion Checklist: Frontend Design

Before using `summary.response.md`, confirm:

- [ ] Approved strategy or discovery brief is present.
- [ ] Audience and page goal are clear.
- [ ] Required sections and known content facts are captured.
- [ ] Brand constraints, references, style preferences, or unknowns are captured.
- [ ] Avoid list is captured or marked none.
- [ ] Design output mode is clear: planning direction, implementation guidance, or visual QA.
- [ ] If direction is open, 2-3 distinctive directions are included.
- [ ] Recommended direction includes rationale.
- [ ] Typography, color, spacing, layout, components, states, motion, and signature visual move are defined when planning.
- [ ] Static mockup requirement and approval status are stated when visual direction needs approval.
- [ ] Vercel-derived interface checks are included where relevant.
- [ ] Anti-slop constraints are listed.
- [ ] Visual QA findings are based on screenshots, rendered page, or local URL evidence when QA is requested.
- [ ] Summary asks the human to approve, modify, or reject.
