# Missing Input Response: Frontend Design

Use this response when frontend design input is incomplete.

Ask only for the missing items that materially affect design direction:

- Approved strategy or discovery brief
- Audience and page goal
- Required sections and content facts
- Brand constraints, references, or style preferences
- Avoid list
- Target stack or implementation context, if implementation guidance is needed
- Screenshots or local URL, if visual QA is needed

If the human has no design preference, say that the skill can propose 2-3 directions and recommend one for approval.

Do not produce final design direction, implementation guidance, or visual QA until blockers are answered or explicitly marked unknown.
